# Ideation ワークフロー（テーマ企画）

記事テーマをネタ出し・評価し、企画を確定させる。
Strategy で定めた方針に基づいてテーマを選定し、Writing（`/article`）へ渡す。

## いつ使うか

- 新しい記事を書き始める前にテーマを決めたいとき
- ネタのストックを増やしたいとき
- 複数のテーマ候補から1つに絞りたいとき

## フロー位置

```
/strategy → /ideation → /article
```

## ワークフロー

### Step 1: ネタ出し

以下を**並列で Read**:
- `01_strategy/03_target/persona.md`（ペルソナ定義）
- **Glob** + **Read** `05_management/02_analysis/*.md`（過去の振り返りシート）
- **Glob** + **Read** `01_strategy/01_calendar/*.md`（コンテンツカレンダー）

テーマ候補を出す方法:
1. **自分の体験・気づきから**: ユーザーに問いかけ
2. **AIインタビュー**: テーマが漠然としている場合は `/interview` を案内
3. **ペルソナの課題から**: persona.md の課題を起点に提案
4. **Strategy のカレンダーから**: カレンダーに沿ったテーマを選定
5. **過去記事の派生**: 振り返りシートから派生テーマを抽出

テーマ候補を `02_planning/01_ideas/` に **Write**。

### Step 2: テーマ評価

1. **Read** [planning_memo.md](planning_memo.md)（テンプレート）
2. テンプレートに沿って評価:
   - ペルソナとの適合度（ペルソナの課題を解決できるか）
   - 自分の独自性（一人称で語れる体験があるか）
   - タイミング（今書くべき理由があるか）
   - 素材の充足度（Step 0 を埋められるだけの素材があるか）
3. **Write** `02_planning/02_evaluation/YYYYMMDD_{テーマ短縮}.md`

### Step 3: テーマ確定

1. **AskUserQuestion** で GO / 保留 / 見送り を確認
2. GO の場合:
   - 企画メモのステータスを「確定」に **Edit** で更新
   - 「`/article {テーマ名}` を実行してください」と案内
3. 保留の場合: 追加素材の収集を提案（`/interview` 等）
4. 見送りの場合: 理由を記録して終了

---

## オプション: ペルソナ検証

テーマの妥当性を検証したい場合は `/persona` スキルを案内。

例: 企画メモのテーマをペルソナに見せて「この記事、読みたいですか？」と聞く。

---

## AI制約

- ネタ出しではAIが提案してよいが、最終的なテーマ選定はユーザーが行う
- 企画メモの「自分の体験・独自性」欄はユーザー本人が記入する。AIが体験を捏造しない

## 関連スキル
