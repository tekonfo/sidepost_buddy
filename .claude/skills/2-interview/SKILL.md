# AIインタビューワークフロー

テーマが漠然としている状態から、AIとの対話を通じて記事素材を具体化する。
出力は Step 0 テンプレート形式でまとめ、そのまま `/article` の素材として使える。

## いつ使うか

- 「このテーマで書きたいけど、何から書けばいいかまとまらない」
- 「体験はあるけど、記事としてどう切り出せばいいかわからない」
- 「漠然としたアイデアはあるが、具体的なエピソードや数字が足りない」
- 「ボイスメモの代わりに、対話で素材を引き出したい」

## フロー位置

```
AIインタビュー → step0_memo.md → /article（Step 0-5）
```

---

## 起動時の処理

1. **Read** [interview_role.md](interview_role.md) でインタビュアー振る舞い定義を読み込み
2. **Read** `01_strategy/03_target/persona.md` を読み込み（読者課題把握用）
3. **TaskCreate** で6要素の充足度を追跡するタスクを作成:
   - テーマの輪郭
   - 具体的な体験
   - 数字・事実
   - コアメッセージ
   - 読者の課題
   - 切り口の独自性
4. インタビュー開始（Claude Code が直接インタビュアー役を演じる）

---

## インタビューの進め方

### 基本方針: 固定フェーズではなく、動的に判断する

チェックリストを内部的に持ち、会話の中でどの要素が足りていないかをリアルタイムに判断して、次の質問を決める。順番は固定しない。

### 記事素材チェックリスト（内部管理）

| 要素 | 十分の基準 | step0での対応セクション |
|------|-----------|----------------------|
| **テーマの輪郭** | 何についての記事か、一文で言える状態 | テーマ |
| **具体的な体験** | 場面・行動・結果が描写できるエピソード1つ以上 | 自分の体験・エピソード |
| **数字・事実** | Before/After、日付、固有名詞など1つ以上 | 数字・事実 |
| **コアメッセージ** | 「この記事で一番言いたいこと」が明確 | 伝えたいこと |
| **読者の課題** | 読者が抱える悩み・壁がイメージできる | その他メモ |
| **切り口の独自性** | 既存記事との差別化ポイントがある | その他メモ |

### 質問の引き出し

[interview_role.md](interview_role.md) に定義された質問集を状況に応じて使う。

---

## 運用ルール

1. **AIは捏造しない**: ユーザーが話していない体験を補完・推測しない
2. **1回の質問は1〜2問まで**: 質問攻めにしない
3. **全体で15〜20分目安**: 足りない要素は「後でボイスメモで追加」でよい
4. **「わからない」はOK**: 「未定」として次へ
5. **有望な要素は即フィードバック**: 「これは記事のフックになりそうです」等
6. **脱線は歓迎**: 有望な方向なら追いかける
7. **チェックリストの中間報告**: 途中で充足度を共有する

---

## 終了処理

全要素が十分に集まった（または時間の目安に達した）ら:

1. 会話内容を step0 テンプレート形式で整理
2. 充足度レポートを作成
3. **Write** 作業フォルダに `step0_memo.md` として保存
4. 不足がある場合は「ボイスメモでこの点を補足すると記事が強くなります」と提案
5. 「`/article` を実行して記事作成を開始できます」と案内

### 出力フォーマット

```markdown
# Step 0: 執筆者メモ（素材）
<!-- AIインタビュー（YYYY-MM-DD）から生成 -->

## テーマ

## 自分の体験・エピソード
（箇条書き。場面・行動・結果をセットで）

## 数字・事実
（曖昧な情報は「未確認」と注記）

## 伝えたいこと

## ボイスメモ書き起こし
<!-- インタビュー後に追加素材があれば貼る -->

## 参考URL・出典

## その他メモ
- 記事の切り口候補:
- 刺さりそうな表現:（ユーザーの発言からピックアップ）
- 想定読者の課題:
- 追加で深掘りしたい点:
```

### 充足度レポート

```markdown
## 素材の充足度

| 数字・事実 | | |
| コアメッセージ | | |
| 読者の課題 | | |
| 切り口の独自性 | | |
```
