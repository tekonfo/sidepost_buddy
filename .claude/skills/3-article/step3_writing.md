## note記事の書き方（固定ルール）

### 全体構成
#### 掴み（冒頭はほぼ固定）
- こんにちは！
- `{状況/前置き}`
- 今回は「{テーマ}」について書きます。
- 結論から言うと、`{結論}` でした。

#### セクション
- 見出しはわかりやすく・端的に書く
- 見出しだけ読んでも内容がわかるようにする

#### 終わりの部分
- 文末の段落は「終わりに」で固定する

### 方針
- 読者を煽らない／誇大にしない

### 文体・表現
- 一人称は「私」
- 口調は「です・ます」
- 専門語句は基本的に使わない（使うなら意味も書く）
  - 例: アサイン -> 任命
  - 例: コンセンサス -> 合意
- 段落の間は1行あける
- （）による補足説明は基本的に使わない
  - 心情描写はOK
- 無駄な言い回しを書かない
  - 「〜することができます」->「〜できます」
  - 「〜について書こうと思います」->「〜を書きます」
- 強調（過剰な強調、煽り、断定調の連発）は使わない

## 推敲チェックリスト

### 読みやすさ
- [ ] 一文が長すぎないか（60文字以内が目安）
- [ ] 専門用語に説明を加えているか
- [ ] 適切に段落を分けているか
- [ ] 箇条書きを効果的に使っているか

### 内容の質
- [ ] 読者の悩みに答えられているか
- [ ] 具体例や根拠を示しているか
- [ ] 読後ゴールに到達できる内容か
- [ ] 差別化ポイントが活きているか

### 文章の質
- [ ] 誤字脱字がないか
- [ ] 表記揺れがないか（「下さい」「ください」など）
- [ ] 同じ表現の繰り返しがないか
- [ ] 冗長な表現を削れないか

### 構成の確認
- [ ] 導入で興味を引けているか
- [ ] 各見出しの内容が見出しと一致しているか
- [ ] 話の流れに違和感がないか
- [ ] まとめで要点と次のアクションを示せているか
