# Analysis ワークフロー（公開後分析）

公開した記事のパフォーマンスを振り返り、次のコンテンツ改善に活かす。

## いつ使うか

- 記事公開後 1週間〜1ヶ月のタイミングで振り返りを行うとき
- 月次でコンテンツ全体のパフォーマンスをまとめるとき
- Strategy の見直し材料を集めるとき

## フロー位置

```
/article（公開後）→ /analysis → /strategy（フィードバック）
```

## ワークフロー

### Step 1: データ収集

1. **Glob** + **Read**（並列）:
   - `03_writing/03_published/` or `03_writing/01_draft/*/step5_publish.md` で公開済み記事情報を取得
   - 引数で指定された作業フォルダの記事情報
2. ユーザーに note ダッシュボードのデータ入力を依頼:
   - PV数（1週間 / 1ヶ月）
   - スキ数
   - コメント数
   - SNSシェア
   - 流入元

### Step 2: 振り返りシート作成

1. **Read** [analysis_sheet.md](analysis_sheet.md)（テンプレート）
2. テンプレートに沿って振り返りシートを作成
3. **Write** `04_analysis/01_data/YYYYMMDD_{記事テーマ短縮}_analysis.md`

### Step 3: 改善点の抽出

1. **Glob** + **Read** 過去の振り返りシート（`04_analysis/01_data/*.md`）
2. パターン分析:
   - テーマ選定の傾向
   - 構成・タイトル・導入の効果
   - 読者反応のパターン
3. 次の記事への具体的なアクションを提案

### Step 4: Strategy へのフィードバック

1. **Glob** + **Read** 最新の中期計画（`01_strategy/02_goals/*.md`）
2. KPI 進捗と計画調整の要否を提示
3. 必要に応じて `/strategy` の実行を案内

---

## 推奨タイミング

| タイミング | やること |
|-----------|---------|
| 公開後 1週間 | 速報値の記録（PV、スキの初動） |
| 公開後 1ヶ月 | 振り返りシートの完成 |
| 月末 | 月間の記事全体を俯瞰し、Strategy へフィードバック |

---

## AI制約

- データの数値はユーザーが入力する。AIは note のダッシュボードにアクセスできないため、データを捏造しない
- 分析・考察はAIが支援してよいが、解釈が主観的な場合は「仮説」と明示する
