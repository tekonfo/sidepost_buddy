# AIインタビューワークフロー

書きたいテーマが漠然としている状態から、AIとの対話を通じて記事の素材を具体化するためのワークフロー。
記事作成ワークフロー（`01_article_creation.md`）の **Step 0（素材）を充実させるための前段プロセス** として使用する。

---

## いつ使うか

- 「このテーマで書きたいけど、何から書けばいいかまとまらない」
- 「体験はあるけど、記事としてどう切り出せばいいかわからない」
- 「漠然としたアイデアはあるが、具体的なエピソードや数字が足りない」
- 「ボイスメモの代わりに、対話で素材を引き出したい」

---

## 既存ワークフローとの関係

```
  AIインタビュー              記事作成ワークフロー
  （本ワークフロー）           （01_article_creation.md）
  ┌───────────────┐          ┌───────────────┐
  │ テーマのタネ    │          │ Step 0: メモ   │
  │  ↓             │  出力 →  │ Step 1: 企画   │
  │ 対話で深掘り    │          │ Step 2: 設計   │
  │  ↓             │          │ Step 3: 執筆   │
  │ step0形式で整理 │          │ Step 4: レビュー│
  └───────────────┘          │ Step 5: 公開   │
                              └───────────────┘
```

- 本ワークフローの出力は **step0テンプレートの形式** でまとめる
- そのまま記事作成ワークフローのStep 0として使える
- ボイスメモでの素材収集と **併用可能**（インタビューで出なかった素材は後から追記してよい）

---

## インタビューの進め方

### 基本方針: 固定フェーズではなく、動的に判断する

AIは以下の「記事素材チェックリスト」を内部的に持ち、会話の中でどの要素が足りていないかをリアルタイムに判断して、次の質問を決める。順番は固定しない。ユーザーの回答に応じて、最も必要な方向に掘り下げる。

### 記事素材チェックリスト（AI内部用）

AIはインタビュー中、以下の充足度を常に評価する。

| 要素 | 十分の基準 | step0での対応セクション |
|---|---|---|
| **テーマの輪郭** | 何についての記事か、一文で言える状態 | テーマ |
| **具体的な体験** | 場面・行動・結果が描写できるエピソードが1つ以上 | 自分の体験・エピソード |
| **数字・事実** | Before/After、日付、固有名詞など具体的データが1つ以上 | 数字・事実 |
| **コアメッセージ** | 「この記事で一番言いたいこと」が明確 | 伝えたいこと |
| **読者の課題** | 読者が抱えている悩み・壁がイメージできる | その他メモ |
| **切り口の独自性** | 既存記事との差別化ポイント、または一人称で語れる角度がある | その他メモ |

### AIの動き方

```
1. ユーザーがテーマのタネを話す
2. AIがチェックリストと照合し、最も弱い要素を特定する
3. その要素を埋めるための質問を1〜2問する
4. ユーザーの回答を受けて、チェックリストを更新する
5. まだ弱い要素があれば → 3に戻る
   有望な脱線を発見 → そちらを追いかける
   全要素が十分 → 整理に進む
```

### 質問の引き出し（AIが状況に応じて使う）

以下はAIが使える質問の引き出し。状況に応じて選ぶ。順番通りに使う必要はない。

**テーマが漠然としているとき**
- そもそも何がきっかけでこの話を書きたいと思った？
- 最近、誰かにこの話をしたことがある？そのときどんな反応だった？
- この話を「誰に」一番聞いてほしい？

**体験・エピソードが薄いとき**
- 一番印象に残っている場面を、映像として思い出すと？
- それをやる前と後で、何が一番変わった？
- 失敗した、うまくいかなかった場面は？

**数字・事実が足りないとき**
- 具体的に「いつ」「何回」「何時間」みたいな数字で言えることは？
- 使ったツールやサービスの名前は？
- 他の方法と比べてどうだった？

**コアメッセージが見えないとき**
- この話を一言でまとめるとしたら？
- 過去の自分に言うとしたら何て言う？
- よくある誤解や「普通はこう思うけど実は違う」ということは？

**切り口が弱いとき**
- 同じテーマの既存記事を読んだことはある？自分の話との違いは？
- 自分だからこそ言えること、自分の立場ならではの視点は？
- 「正解」ではなく「自分はこうやっている」で語れる部分は？

**話が広がりすぎているとき**
- ここまでの話の中で、一番「これを伝えたい」と感じたのはどこ？
- 1記事にするなら、どの部分に絞る？

## 運用ルール

1. **AIは捏造しない**: インタビューで引き出した内容のみを使用する。ユーザーが話していない体験をAIが補完・推測しない
2. **1回の質問は1〜2問まで**: 質問攻めにしない。ユーザーが答えやすいテンポを維持する
3. **全体で15〜20分目安**: 長引きすぎない。足りない要素があっても「ここは後でボイスメモで追加」でよい
4. **「わからない」はOK**: ユーザーが「まだ明確じゃない」と答えたら、そのまま記録し別の角度から聞く。それでも出なければ「未定」として次へ
5. **有望な要素は即フィードバック**: 答えの中から記事として有望な要素が見えたら、その場で「これは記事のフックになりそうです」等と指摘してよい
6. **脱線は歓迎**: 予定外の方向に話が広がった場合、それが有望なら追いかける
7. **チェックリストの中間報告**: インタビューの途中で「ここまでで○○は十分集まりました。△△がもう少し欲しいです」と現在地を共有する

---

## 整理（インタビュー終了時）

全要素が十分に集まった（または時間の目安に達した）ら、AIが会話内容を **step0テンプレートの形式** で整理する。

### 出力フォーマット

```markdown
# Step 0: 執筆者メモ（素材）
<!-- AIインタビュー（YYYY-MM-DD）から生成 -->

## テーマ


## 自分の体験・エピソード
（箇条書き。場面・行動・結果をセットで）

## 数字・事実
（曖昧な情報は「未確認」と注記）

## 伝えたいこと


## ボイスメモ書き起こし
<!-- インタビュー後に追加素材があれば貼る -->

## 参考URL・出典


## その他メモ
- 記事の切り口候補:
- 刺さりそうな表現:（ユーザーの発言からピックアップ）
- 想定読者の課題:
- 追加で深掘りしたい点:（インタビューで埋まらなかった部分）
```

### 充足度レポート

整理と合わせて、チェックリストの最終状態も提示する。

```markdown
## 素材の充足度
| 要素 | 状態 | メモ |
|---|---|---|
| テーマの輪郭 | ✅ 十分 / ⚠️ やや弱い / ❌ 不足 | |
| 具体的な体験 | | |
| 数字・事実 | | |
| コアメッセージ | | |
| 読者の課題 | | |
| 切り口の独自性 | | |
```

不足がある場合は「ボイスメモでこの点を補足すると記事が強くなります」等、次のアクションを提案する。

## AIへの指示（プロンプト）

以下のプロンプトでインタビューを開始する。

```
あなたはアナウンサーの有働由美子さんのインタビュースタイルで、私から記事の素材を引き出す編集者です。

## 有働さんのインタビュースタイル
- 相手の言葉を受け止めてから、核心に踏み込む。「なるほど。でもそれって本当にそうですか？」のように、柔らかく、でも遠慮しない
- 表面的な回答で終わらせない。「もうちょっと聞いていいですか」「それ、具体的にはどういうこと？」と自然に深掘りする
- 相手が言語化できていないことを代わりに言語化してみせる。「つまりこういうことですか？」と投げかけて、本人の言葉を引き出す
- 共感はするが、甘やかさない。「わかります。でも読者はそこが一番知りたいと思うんですよ」と記事のゴールに引き戻す
- テンポがいい。長い前置きや過剰な褒めはしない。聞いて、受けて、次に行く

## あなたの役割
- 私が書きたいテーマについて、対話で素材を引き出す
- 漠然とした考えを、具体的なエピソード・数字・メッセージに落とし込む手助けをする
- 記事の切り口や読者への刺さりポイントも一緒に考える

## 記事素材チェックリスト
インタビュー中、以下の要素の充足度を常に評価し、足りない部分を重点的に聞く。
- テーマの輪郭（何についての記事か一文で言える）
- 具体的な体験（場面・行動・結果が描写できるエピソード）
- 数字・事実（Before/After、日付、ツール名など）
- コアメッセージ（この記事で一番言いたいこと）
- 読者の課題（読者が抱えている悩み・壁）
- 切り口の独自性（自分だから語れる角度）

## 進め方
1. まずテーマのタネを聞く
2. 回答に応じて、チェックリストで最も弱い要素を特定し、そこを掘り下げる
3. 順番は固定しない。会話の流れに沿って自然に進める
4. 有望な脱線は追いかける
5. 全要素が十分に集まったら、step0形式で整理する

## ルール
- 1回の質問は1〜2問まで（質問攻めにしない）
- 私の答えを捏造・補完しない
- 有望な要素が見えたら、「あ、今の話すごくいいですね。それ記事の入口になりますよ」のように指摘してよい
- 途中で「ここまでの充足度」を共有してよい
- 表面的な回答には「もう一歩踏み込んで聞きたいんですけど」と切り込む

それでは始めます。今日はよろしくお願いします。早速なんですけど、書きたいなと思っているテーマ、ざっくりでいいので教えてもらえますか？
```

## ペルソナ会話との使い分け

| | AIインタビュー（本ワークフロー） | ペルソナ会話シミュレーション |
|---|---|---|
| **目的** | 自分の中にある素材を引き出す | 読者がどう反応するかを検証する |
| **AIの役割** | 編集者（質問する側） | ペルソナ（答える側） |
| **ユーザーの役割** | 答える側（体験を語る） | 質問する側（コンテンツを検証する） |
| **使うタイミング** | テーマはあるが素材が足りないとき | 記事案や企画がある程度固まったとき |
| **出力** | step0（執筆者メモ） | インサイト・記事案リスト |

**推奨フロー**: AIインタビューで素材を引き出す → 記事案をまとめる → ペルソナ会話で読者反応を検証する

---

## 応用: 複数記事のネタ出しインタビュー

1つのテーマから複数記事に展開したい場合は、整理時に以下の観点も追加する。

- この体験から、切り口を変えて何本の記事が書けそうか？
- 読者の悩みの段階（知らない→知ってるけどできない→できるけど続かない）で分けると？
- 初心者向け・経験者向けで記事を分けるなら？

出力は記事案リスト形式で整理し、`03_writing/01_draft/` 配下に保存する。
