# Analysis ワークフロー（公開後分析）

公開した記事のパフォーマンスを振り返り、次のコンテンツ改善に活かすワークフロー。

---

## いつ使うか

- 記事公開後 1週間〜1ヶ月のタイミングで振り返りを行うとき
- 月次でコンテンツ全体のパフォーマンスをまとめるとき
- Strategy の見直し材料を集めるとき

---

## 既存ワークフローとの関係

```
  Writing               Analysis               Strategy
  （01_article_creation） （本ワークフロー）       （09_strategy.md）
  ┌──────────────┐      ┌──────────────┐       ┌──────────────┐
  │ 記事公開      │      │ データ収集    │       │ 方針見直し    │
  │ （Step 5）    │ →    │ 振り返り      │  →    │ カレンダー    │
  │              │      │ 改善点抽出    │       │ 調整         │
  └──────────────┘      └──────────────┘       └──────────────┘
```

## ワークフロー

### Step 1: データ収集

記事公開後、以下の指標を収集する:

- **PV数**: 公開後 1週間 / 1ヶ月
- **スキ数**: 公開後 1週間 / 1ヶ月
- **コメント**: 内容と傾向
- **SNSシェア**: 反応の有無・内容
- **流入元**: 検索 / SNS / note内回遊
- **読了率**: note の分析機能で確認（可能な場合）

### Step 2: 振り返りシート作成

1. `00_config/template/analysis_sheet.md` をコピーして `04_analysis/01_data/` に保存
2. ファイル名: `YYYYMMDD_{記事テーマ短縮}_analysis.md`
3. 以下を記入する:
   - 記事の基本情報（タイトル、公開日、URL）
   - 定量データ（PV、スキ、コメント数等）
   - 定性分析（読者の反応、想定との差異）
   - 良かった点 / 改善点
   - 次回への学び

### Step 3: 改善点の抽出

1. 振り返りの結果から、再現すべきこと・改善すべきことを整理する
2. テーマ選定・構成・タイトル・導入・CTA など観点別に分析する
3. 次の記事やStrategy への具体的なアクションを記録する

### Step 4: Strategy へのフィードバック

1. 月次で振り返りシートを集約し、傾向を把握する
2. KPI進捗を確認し、Strategy の計画を調整する必要があるか判断する
3. 必要に応じて `01_strategy/` の計画を更新する

---

## 推奨タイミング

| タイミング | やること |
|-----------|---------|
| 公開後 1週間 | 速報値の記録（PV、スキの初動） |
| 公開後 1ヶ月 | 振り返りシートの完成 |
| 月末 | 月間の記事全体を俯瞰し、Strategy へフィードバック |

---

## AI制約

- データの数値はユーザーが入力する。AIはnoteのダッシュボードにアクセスできないため、データを捏造しない
- 分析・考察はAIが支援してよいが、解釈が主観的な場合は「仮説」と明示する
