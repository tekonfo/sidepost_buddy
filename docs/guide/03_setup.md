# Chapter 3: 初期セットアップ

記事を書き始める前に、3 つの設定を用意します。すでに設定済みの場合はスキップして構いません。

## 1. ペルソナの定義

**ペルソナ**は「想定読者のプロフィール」です。記事設計やレビューの基準になります。

### 設定ファイル

```
01_strategy/03_target/persona.md
```

### 記載項目

ペルソナは最大 3 名分を定義できます。各ペルソナには以下の情報を書きます。

| 項目 | 記載例 |
|------|--------|
| 基本情報 | 35歳、会社員、子ども1人 |
| 状況 | 副業を始めたいが何から手をつけていいかわからない |
| 悩み・課題 | 時間がない、失敗が怖い、情報が多すぎる |
| お金・副業に対する考え方 | 月3万円でいいから安定した副収入がほしい |
| 影響を受けている情報源 | YouTube、Twitter、note |
| AI ツールへの馴染み | ChatGPT は使ったことがある程度 |
| 過去の失敗体験 | ブログを始めたが3記事で挫折 |
| 記事に期待すること | 具体的な手順、実体験ベースのアドバイス |

### やり方

1. `01_strategy/03_target/persona.md` を開く
2. テンプレートの各項目を埋める（最低 1 名分）
3. 実在の知人や過去の自分をモデルにするとリアルに書ける

> **ヒント**: ペルソナが思い浮かばない場合は `/persona` コマンドで AI と対話しながら具体化できます。

## 2. ブランドスクリプトの作成

**ブランドスクリプト**は、StoryBrand SB7 フレームワークに基づく「発信の軸」です。

### 設定ファイル

```
01_strategy/04_brand/brand_script.md
```

### 7 つの要素

| # | 要素 | 質問 |
|---|------|------|
| 1 | Character（主人公） | 読者は誰か？何を望んでいるか？ |
| 2 | Problem（課題） | 読者が直面している問題は？ |
| 3 | Guide（導き手） | あなたはどんな立場で語るか？ |
| 4 | Plan（計画） | 読者に提示するステップは？ |
| 5 | CTA（行動喚起） | 読者にしてほしいアクションは？ |
| 6 | Failure（失敗） | 行動しないとどうなるか？ |
| 7 | Success（成功） | 行動するとどうなるか？ |

### やり方

1. `01_strategy/04_brand/brand_script.md` を開く
2. テンプレートに沿って 7 つの要素を埋める
3. 迷ったら `01_strategy/04_brand/brand_script_template.md` の記入例を参考にする

## 3. トーン＆マナーの設定

**トーン＆マナー**は記事の文体ルールです。記事間で一貫した書き方を保つために使います。

### 設定ファイル

```
00_config/concept/tone_manner.md
```

### 主な設定項目

| カテゴリ | 内容例 |
|---------|--------|
| 基本トーン | 一人称「私」、敬体（です/ます） |
| 使う表現 | 「〜してみてください」「実際にやってみたところ」 |
| 避ける表現 | 「絶対に」「〜しなければ損」 |
| 記事の構成ルール | 冒頭は「こんにちは！」から、末尾は「終わりに」 |
| やらないこと | 過度な煽り、根拠のない断言 |

### やり方

1. `00_config/concept/tone_manner.md` を開く
2. 自分の書きたい文体に合わせて各項目を記入する
3. 既存の note 記事があれば、そのトーンを参考にすると一貫性が保てる

## セットアップの確認

3 つのファイルが準備できたか確認しましょう。

```
✅ 01_strategy/03_target/persona.md     ... ペルソナが1名以上定義済み
✅ 01_strategy/04_brand/brand_script.md  ... 7要素が記入済み
✅ 00_config/concept/tone_manner.md      ... トーン設定が記入済み
```

> **すべて完璧でなくて OK です。** 最初はざっくりでも書いておけば、記事を書きながら徐々に精度を上げていけます。
