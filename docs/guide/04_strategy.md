# Chapter 4: Strategy フェーズ

## このフェーズの目的

「何を、いつ、どれくらい書くか」の中期計画を立てるフェーズです。

- コンテンツの方向性を決める
- 月ごとのテーマ配分を計画する
- KPI（投稿数・PV・フォロワー等）の目標を設定する

## コマンド

```
/strategy
```

引数: `new`（新規作成）または `update`（既存計画の見直し）

## 成果物

| ファイル | 配置場所 | 内容 |
|---------|---------|------|
| 戦略シート | `01_strategy/02_goals/` | 目的・テーマ・KPI・差別化ポイント |
| コンテンツカレンダー | `01_strategy/01_calendar/` | 月別のテーマ配分 |

## ワークフロー

### 1. 現状分析

AI が以下の情報を読み込みます。

- ペルソナ定義（`01_strategy/03_target/persona.md`）
- ブランドスクリプト（`01_strategy/04_brand/brand_script.md`）
- 過去の分析結果（あれば）

### 2. 方針の策定

対話形式で以下を決めていきます。

- **コンテンツの目的**: なぜ記事を書くのか
- **メインテーマ / サブテーマ**: 何について書くのか
- **差別化ポイント**: 他の発信者との違い
- **KPI 目標**: 月間投稿数、PV 目標、フォロワー目標 など

### 3. カレンダー作成

月ごとにテーマの配分を計画します。

```
例:
3月: 副業の始め方（2本）、AI活用（1本）
4月: 時間管理（2本）、副業の始め方（1本）
```

### 4. 確認・承認

計画をユーザーが確認し、必要に応じて修正します。
オプションで `/persona` を使い、ペルソナ視点から計画の妥当性を検証できます。

## 初めての場合のコツ

- **まずは 1〜2 ヶ月分**で十分。長期計画は記事を何本か書いてからの方が精度が上がる
- **KPI は控えめに**設定する。月 2 本から始めて、慣れたら増やすのがおすすめ
- **テーマは 2〜3 カテゴリ**に絞る。広げすぎると方向性がぼやける
