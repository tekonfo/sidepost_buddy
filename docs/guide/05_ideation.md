# Chapter 5: Planning フェーズ

## このフェーズの目的

「次に何を書くか」を決めるフェーズです。テーマのアイデアを出し、評価し、記事として書くかどうかを判断します。

## コマンド

```
/ideation
```

※ コマンド名は互換性のため `/ideation` のままです。

## テーマの発想方法

アイデアの出し方は複数あります。組み合わせて使うと効果的です。

| 方法 | やり方 |
|------|--------|
| 自分の体験から | 最近やったこと、うまくいったこと、失敗から学んだこと |
| ペルソナの悩みから | ペルソナ定義の「悩み・課題」を見て、答えになるテーマを考える |
| カレンダーから | Strategy で計画したテーマ配分を確認する |
| 過去記事の派生 | 公開済み記事の反応が良かったテーマを深掘りする |
| AI インタビュー | `/interview` コマンドで対話しながら素材を引き出す |
| デイリーノートから | `/daily-note-article-items` で日常メモから種を拾う |

## テーマの評価

アイデアが出たら、**企画メモ**（planning_memo）で評価します。

### 評価の 4 軸

| 軸 | 確認すること |
|----|-------------|
| **ペルソナ適合度** | ターゲット読者の悩み・関心に合っているか |
| **オリジナリティ** | 自分の体験や知見を活かせるか |
| **素材の充実度** | Step 0 に書ける体験・事実・数字があるか |
| **タイミング** | 今出す意味があるか（季節性、トレンド等） |

### 判定

| 判定 | 意味 | 次のアクション |
|------|------|---------------|
| **GO** | 書ける | → `/article` で記事作成を開始 |
| **Hold** | もう少し素材が必要 | → 素材を集めてから再評価 |
| **Skip** | 今は書かない | → 理由を記録して次のテーマへ |

## `/interview` コマンド（AI インタビュー）

テーマが漠然としているとき、AI がインタビュアーとして対話し、記事の素材を引き出します。

```
/interview
```

### 引き出す情報

1. テーマの概要
2. 具体的な体験エピソード
3. 数字・事実
4. 伝えたい核心メッセージ
5. 読者が抱えている問題
6. 独自の切り口

### 出力

インタビュー結果は `step0_memo.md` 形式で出力されるため、そのまま `/article` に持ち込めます。

## 企画メモの保存場所

```
02_planning/02_evaluation/YYYYMMDD_テーマ名.md
```

## 初めての場合のコツ

- **完璧なテーマを探さない**。「自分が体験したこと」が書けるなら、それで十分
- **素材の充実度を最優先**する。いい切り口でも書くネタがなければ苦しくなる
- **迷ったら `/interview`** を試す。対話するうちに「これなら書ける」というテーマが見つかることが多い

## 次のステップ
