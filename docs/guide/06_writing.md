# Chapter 6: Writing フェーズ

記事作成の中心となるフェーズです。6 つのステップとゲートチェックを順に進めます。

## コマンド

```
/article テーマ名
```

新規作成の場合はテーマ名を指定します。途中から再開する場合は:

```
/article resume
```

## 全体の流れ

```
Step 0 → [Gate 1] → Step 1 → Step 2 → [Gate 2] → Step 3 → Step 4 → [Gate 3] → Step 4.5 → Step 5
```

---

## Step 0: 素材メモ

**ファイル**: `step0_memo.md`

記事のすべての土台になるステップです。**あなた自身の体験・事実・考え**を書きます。

### 書く内容

| 項目 | 説明 | 例 |
|------|------|----|
| テーマ | 何について書くか | 「副業を始めて3ヶ月で月3万円稼いだ方法」 |
| 体験エピソード | 具体的な場面 + 行動 + 結果 | 「最初の案件は500円のライティング。不安だったが...」 |
| 数字・事実 | 具体的なデータ | 「3ヶ月で計15件受注、合計32,000円」 |
| 伝えたいこと | 記事の核心メッセージ | 「小さく始めることが継続のコツ」 |
| 参考 URL | 引用したい情報源 | リンクやメモ |

### 重要なルール

> **AI は Step 0 の内容を捏造・補完・推測してはいけません。**
> ここに書かれるのはあなたの実体験のみです。AI は整形・校正のみ行います。

### Gate 1

以下を満たしたら次へ進めます。

- テーマが明記されている
- 体験エピソードが 1 つ以上ある

---

## Step 1: リサーチ

**ファイル**: `step1_research.md`

AI が Step 0 の内容をもとに関連情報を調査します。

- 類似記事の調査
- キーワードリサーチ
- 読者が求めている情報の分析

ユーザーの作業は基本的に不要です。結果を確認して、方向性のズレがあれば指摘してください。

---

## Step 2: 記事設計

**ファイル**: `step2_design.md`

記事の構成（目次）を設計します。

### やること

1. **ペルソナを 1 名選ぶ**: この記事は誰に向けて書くか
2. **見出し構成を作る**: H2/H3 レベルの目次
3. **不足情報の洗い出し**: 記事に必要だが Step 0 にない情報を特定

### 不足情報の扱い

| レベル | 意味 | 対応 |
|--------|------|------|
| **必須** | これがないと記事が成り立たない | → ユーザーに追加入力を依頼 |
| **あると良い** | あればより良い記事になる | → なくても先に進める |

### Gate 2

- 必須レベルの不足情報がすべて解消されていること

---

## Step 3: 本文執筆

**ファイル**: `step3_writing.md`

AI が Step 0〜2 の情報をもとに記事本文を書きます。

### note 記事のルール

以下のルールに沿って執筆されます（トーン＆マナー設定に準拠）。

| ルール | 内容 |
|--------|------|
| 冒頭 | 「こんにちは！」→ 背景 → テーマ → 結論 |
| 一人称 | 「私」 |
| 文体 | です/ます調 |
| 専門用語 | 使わない（使う場合は説明を添える） |
| 見出し | ひと目で内容がわかる簡潔な表現 |
| 末尾 | 「終わりに」で締める |

### ユーザーの確認ポイント

- 事実と異なる記述がないか
- 伝えたいニュアンスとズレがないか
- 追加したいエピソードや表現がないか

修正したい箇所があれば、具体的に指示すれば AI が修正します。

---

## Step 4: ペルソナレビュー

**ファイル**: `step4_review.md`

AI がペルソナ（想定読者）の視点で記事を評価します。

### 評価の 5 軸

| 軸 | 確認すること |
|----|-------------|
| **共感** | 冒頭で「自分のための記事だ」と感じるか |
| **理解** | 専門用語なしで理解できるか |
| **説得力** | 根拠・具体例で納得できるか |
| **行動明確性** | 読後に何をすればいいかわかるか |
| **信頼性** | 煽りや誇張がないか |

### 判定基準

各軸を ○（OK）/ △（軽微な改善）/ ×（要修正）で評価します。

| 合格条件 | × = 0 かつ △ ≤ 2 |
|---------|-------------------|
| 不合格の場合 | 指摘をもとに修正 → 再レビュー（最大 3 ラウンド） |

### Gate 3

レビュー合格条件を満たしたら次へ。

---

## Step 4.5: スライド画像（任意）

記事に挿入するスライド画像を生成します。必須ではありません。

```
/slide ジョブフォルダ名
```

詳しくは [Chapter 8: 便利機能ガイド](./08_utilities.md) を参照してください。

---

## Step 5: 公開準備

**ファイル**: `step5_publish.md`

最終チェックリストを確認して公開に備えます。

- タイトルの最終確認
- 本文の最終確認
- 画像の確認
- note への投稿設定（カテゴリ、タグ等）

ここまで完了したら、note にコピー＆ペーストして公開します。

---

## 途中で中断・再開する場合

各ステップの完了時に `progress.md` が自動更新されます。次のセッションで `/article resume` を実行すると、中断したステップから再開できます。

## 初めての場合のコツ

- **Step 0 に時間をかける**。素材が充実していれば、その後のステップはスムーズ
- **完璧を目指さない**。Step 3 の初稿は 70% の出来で OK。Step 4 のレビューで磨かれる
- **修正指示は具体的に**。「もっと良くして」より「この段落に◯◯のエピソードを追加して」の方が結果が良い
