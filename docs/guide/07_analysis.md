# Chapter 7: Management フェーズ

## このフェーズの目的

公開した記事のパフォーマンスを振り返り、次の Strategy にフィードバックする管理フェーズです。

## コマンド

```
/analysis
```

※ コマンド名は互換性のため `/analysis` のままです。

## 振り返りのタイミング

| タイミング | やること |
|-----------|---------|
| **公開 1 週間後** | 速報チェック（PV・いいね の初動） |
| **公開 1 ヶ月後** | 本格的なレビューシートの作成 |
| **月末** | その月の記事をまとめて振り返り → Strategy へフィードバック |

## ワークフロー

### 1. データ収集

AI が以下の情報をヒアリングします（note のダッシュボードから確認）。

| 指標 | 説明 |
|------|------|
| PV | 1 週間 / 1 ヶ月のページビュー |
| いいね数 | 記事への「スキ」の数 |
| コメント | 読者からのコメント内容 |
| 流入元 | どこから読者が来たか（検索、SNS、note内） |

### 2. レビューシートの作成

収集したデータをもとに分析シートを作成します。

- 目標（KPI）との比較
- うまくいった点
- 改善すべき点
- 読者の反応から見えた傾向

### 3. パターン分析

過去のレビューシートがあれば、複数記事の傾向を分析します。

- どのテーマが反応が良いか
- どの時間帯・曜日に PV が伸びるか
- いいねが多い記事の共通点

### 4. Strategy へのフィードバック

分析結果をもとに、次の Strategy に向けた提案を行います。

- テーマ配分の調整
- KPI 目標の修正
- 新しいテーマの候補

## 成果物の保存場所

```
05_management/02_analysis/
```

## 初めての場合のコツ

- **最初の数記事は「データを貯める期間」** と割り切る。1 記事で傾向は見えない
- **数字に一喜一憂しない**。PV が少なくても、いいねの比率が高ければ質の良い記事
- **定性的なフィードバック**（コメント、引用RT等）も重要な材料

## 次のステップ

分析結果をもとに `/strategy update` で計画を見直すと、フェーズが一巡します。
便利機能について知りたい場合は [Chapter 8: 便利機能ガイド](./08_utilities.md) へ。
