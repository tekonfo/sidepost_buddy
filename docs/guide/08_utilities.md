# Chapter 8: 便利機能ガイド

メインの 5 フェーズ以外にも、記事作成を支援する機能があります。

---

## `/persona` — ペルソナ会話シミュレーション

ペルソナ（想定読者）になりきった AI と対話し、コンテンツの反応をシミュレーションします。

### 使いどころ

- テーマ案が読者に刺さるか確認したい
- タイトルの候補を比較したい
- 記事を読んだ感想を聞きたい
- 有料記事にした場合の購買意欲を測りたい

### 使い方

```
/persona
```

引数でペルソナ番号（1〜3）を指定できます。省略すると選択を促されます。

### 出力内容

| 項目 | 説明 |
|------|------|
| 刺さったポイント | 共感・興味を引いた点 |
| 刺さらなかったポイント | 響かなかった・違和感があった点 |
| 新たに見えた課題 | シミュレーションで気づいた問題 |
| コンテンツへの示唆 | 改善の方向性 |

---

## `/slide` — スライド画像生成

記事に挿入するスライド画像を YAML 定義から自動生成します。

### 前提条件

- Python 環境がセットアップ済みであること
- Gemini API キーが設定済みであること

### 使い方

```
/slide ジョブフォルダ名
```

### ワークフロー

1. **YAML 編集**: `image_assets/slide_image_prompts.yaml` に画像の定義を記述
2. **構文チェック**: YAML の文法を検証
3. **画像生成**: Gemini API でスライド画像を生成

### パターンテンプレート

| テンプレート | 用途 |
|-------------|------|
| `00_rule_and_blank.yaml` | デフォルト（ルール + 空白） |
| `01_flow.yaml` | フロー図 |
| `02_comparison.yaml` | 比較表 |
| `03_timeline.yaml` | タイムライン |
| `04_funnel.yaml` | ファネル図 |

---

## `/screenshot` — スクリーンショット加工

スクリーンショットにプライバシー保護加工（ぼかし + ハイライト）を施します。

### 使いどころ

- 画面キャプチャに個人情報が映っている
- 注目してほしい部分を強調したい

### 使い方

```
/screenshot
```

### 処理内容

1. AI が画像を OCR で解析し、機密情報の位置を検出
2. 機密部分にガウスぼかしを適用
3. 注目範囲に赤枠ハイライトを追加
4. ユーザーが結果を確認・調整

---

## `/review` — ペルソナレビュー単体実行

Writing フェーズの Step 4（ペルソナレビュー）を単独で実行します。

### 使いどころ

- `/article` ワークフロー外で既存の記事をレビューしたい
- 修正後に特定のペルソナで再レビューしたい
- 別のペルソナで追加レビューしたい

### 使い方

```
/review
```

---

## `/interview` — AI インタビュー

テーマが漠然としているとき、プロインタビュアー風の AI が対話で素材を引き出します。

### 使いどころ

- 「何か書きたいけどテーマが固まらない」
- 体験はあるが、記事としてどう切り取るか迷っている

### 使い方

```
/interview
```

### 引き出す 6 要素

1. テーマの概要
2. 具体的な体験エピソード
3. 数字・事実
4. 核心メッセージ
5. 読者が抱えている問題
6. 独自の切り口

出力は `step0_memo.md` 形式なので、そのまま `/article` に引き継げます。

---

## `/daily-note-article-items` — デイリーノートから素材抽出

Obsidian のデイリーノートから記事に使える素材を自動抽出します。

### 使いどころ

- 日常のメモから記事のネタを拾いたい
- Step 0 の素材を過去のメモから集めたい

### 使い方

```
/daily-note-article-items
```

### 抽出する情報

| カテゴリ | 説明 |
|---------|------|
| テーマ候補 | 記事テーマになりそうな話題 |
| 体験エピソード候補 | 記事に使える具体的な出来事 |
| 数字・事実候補 | データや検証結果 |
| メッセージ候補 | 伝えたいことの種 |

---

## `/overview` — 全体マップ表示

ライフサイクル全体の状態を俯瞰し、進行中の記事を検出します。

### 使い方

```
/overview
```

### 表示内容

- 5 フェーズのライフサイクルマップ
- 進行中の記事一覧と現在のステップ
- 次にやるべきアクションの提案

セッション開始時に現在地を確認するのに便利です。
