こんな悩み、ありませんか？

Case 01: 手が止まる

「書くこと」がない
情報の解像度不足

  • いざ書こうとしても、中身が薄くて筆が進まない
  • 「何を伝えるべきか」の軸がブレてしまう
  • 下書きが10個以上溜まったまま、中途半端に放置される

Case 02: 準備不足

いきなり「執筆」に
取り掛かってしまう

  • 料理と同じ。下準備なしで名作は生まれない
  • 自分のリアルな体験や調査データが不足している
  • AIに丸投げして、内容のない「AI感」満載の文になる

Case 03: 管理の欠如

「書きっぱなし」で
資産にならない

  • 過去の記事がどう貢献しているか分からない
  • 伝えたいコンセプトが全体で一貫していない
  • 執筆ステータスが管理できず、重要なネタを忘れる
これらの悩みは、あなたの才能の問題ではありません。 「企画・実行・執筆」の分離と、それを支える「管理」の仕組みが必要です。

PHASE 01

戦略策定 (Strategy)

発信の「OS」を構築し、AIバディにあなたの価値観をインストールする最重要フェーズです。

成果物: persona.md

ペルソナ定義

👤

単なる属性情報ではなく、読者が夜も眠れないほど悩んでいる「痛み」と、記事を読んだ後の「理想の状態」を定義します。

活用シーン:
  • AIによるターゲット視点の記事レビュー
  • 読者の悩みに突き刺さる「切り口」の生成基準
成果物: brand_script.md

ブランド・スクリプト

🎯

読者を「主人公」とし、あなたを「導き手」とする物語を設計。発信の一貫性と独自価値を言語化します。

設定内容:
  • 独自価値: あなたにしか語れない解決策
  • 成功の定義: 読者が得られる究極のベネフィット
  • トンマナ: 信頼と共感を生む言葉選びの指針
PHASE 02

管理する (Management)

企画・実行・執筆のサイクルを最適化し、発信を停滞させないための管制塔です。

成果物: article_progress.csv

進捗の棚卸し

📋

「書きたい」と思ったネタがどこで止まっているかを可視化。中途半端な下書きを救出し、完了へと導きます。

管理項目:
  • 企画・実行・執筆の各ステータス管理
  • 公開済み記事の資産化と再利用の検討
分析・改善

反応の分析と還元

📈

公開後の反応（PV/スキ数）を分析し、戦略や次の企画へ還元。読者の真のニーズに合わせて発信の精度を高めます。

フィードバック内容:
  • ターゲットの悩みに適合していたかの検証
  • 次に優先して執筆すべきテーマの特定
プロセス改善

振り返りと最適化

🔄

「記事を何本出したか」という定量的な実績に加え、「なぜ筆が止まったのか」という定性的なボトルネックを徹底的に分析します。

振り返りの視点:
  • 実績管理: 期間内の公開本数と目標達成度の確認
  • 詰まりの解消: 「調査が重すぎた」「企画が曖昧だった」等の停滞原因の特定
  • 仕組みの更新: 振り返りを元に、プロンプトやワークフローを改善
STEP 01

企画する (Planning)

日々の「種」を拾い上げ、戦略に沿った価値ある企画へ昇華させるプロセスです。

プロセス: 01

ネタの自動抽出

🌱

日々のデイリーノートから、記事になりそうな「思考の断片」をAIが自動でスクレイピングします。

仕組み:
  • ノート内の特定のタグや記述から独自の気づきを特定
  • 未整理のメモを「企画の種」としてプール
プロセス: 02

評価マトリックス

📊

抽出したネタを客観的な指標でスコアリング。本当に書く価値のあるものだけを厳選します。

評価項目:
  • 需要: 読者の痛みを解決できるか？
  • 独自性: あなたならではの体験が含まれるか？
  • 納得感: 読者が納得できる解決策を提示できるか？
プロセス: 03

AIインタビュー

🎤

企画が決まったら、AIが深掘り質問を実施。あなたの頭の中にある「暗黙知」を引き出し、構成の核を作ります。

STEP 02

仕込む (Prep)

「下調べ」と「試行」のフェーズ。執筆を加速させるための最高品質の素材を揃えます。

プロセス: 01

情報収集・調査

🔍

企画テーマに関連する市場の反応や競合記事を調査。何が語られ、何が語られていないかを特定します。

実施内容:
  • 事実情報の裏付け、エビデンスの収集
  • 読者のナマの声、SNS等の反応調査
プロセス: 02

体験と試行の記録

🛠️

実際に手を動かし、試行錯誤の過程を `notes` フォルダに記録。成功だけでなく「失敗のプロセス」こそが独自価値になります。

蓄積される素材:
  • スクリーンショット、検証データ、その時の生々しい感情
  • 独自の工夫点、意外な発見のメモ
STEP 03

執筆する (Writing)

揃った素材を、AIとの共創によって一気に記事へと昇華させるフィニッシュフェーズです。

ワークフロー

6ステップ・パイプライン

🚀

`/article` コマンドを起点とする標準化されたプロセス。迷いを排除し、執筆の摩擦をゼロにします。

ステップ詳細:
  • 0.メモ: 企画・実行の情報を集約
  • 1.調査 / 2.設計: 構成案の確定
  • 3.執筆 / 4.推敲 / 5.公開: 記事の完成
手法

執筆パターンの使い分け

📝

内容の性質に応じて、AIとの共創パターンを切り替えます。

パターン:
  • ボトムアップ: 散らばったメモから共通項を見出し、記事化
  • トップダウン: 構成案を固め、各所に素材を埋め込み
