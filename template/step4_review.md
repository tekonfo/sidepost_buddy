# Step 4: 記事レビュー（subagent用）

このステップでは、完成した記事を**step2で選定した1人のペルソナ**でsubagentに渡してレビューさせる。
ターゲットペルソナに絞ることで、記事が最も届けたい読者に刺さるかを集中的に検証する。
**合格基準を満たすまで「修正→再レビュー」を繰り返す。**

---

## 合格基準

以下をすべて満たした場合に合格（step5へ進める）。

1. **×（要修正）がゼロ**: レビューで×が1つもないこと
2. **△が2件以下**: 5観点中、△が2件以下であること

満たさない場合は「修正 → 再レビュー」のサイクルを回す（最大3回）。
3回で合格しない場合は、残課題を記録してユーザー判断を仰ぐ。

---

## レビュー前の準備

### レビュー対象ペルソナの特定

step2_design.md の「ペルソナ」セクションで選定したペルソナを使用する。

### subagentに渡す情報

以下の3点をsubagentに入力として渡すこと。

1. **ペルソナ情報**（`01_setting/ペルソナ.md` から**step2で選定したペルソナ**のセクション全体を参照）

2. **記事本文**（step3_writing.md の本文）

3. **ブランドトンマナ**（`01_setting/ブランドトンマナ.md` のセクション2「トーン」を参照）

### 実行方法
- subagentを1つ起動し、選定ペルソナを割り当てる
- 結果を本ファイルの「レビュー結果の記録」にまとめる

---

## subagentへの指示プロンプト

```
あなたは以下のペルソナになりきって、記事をレビューしてください。

【ペルソナ】
`01_setting/ペルソナ.md` の「ペルソナX（該当者名）」セクションにある「基本情報」「状況」「悩み・課題」「AIツールとの関係」「読後に期待するゴール」「刺さるコンテンツの傾向」を読み込んでください。

【レビュー観点】
以下の5つの観点で記事を評価し、それぞれ「○（問題なし）/ △（改善余地あり）/ ×（要修正）」と具体的なコメントを返してください。

1. 共感度：冒頭で「自分のための記事だ」と感じられるか
2. 理解度：専門用語や前提知識なしで内容を理解できるか
3. 納得度：主張に根拠・具体例があり、納得できるか
4. 行動度：読後に何をすればよいか明確か
5. 信頼度：煽りや誇張がなく、信頼できる文章か

【出力フォーマット】

## ペルソナX：（名前）のレビュー

| 観点 | 評価 | コメント |
|------|------|----------|
| 共感度 | ○/△/× | |
| 理解度 | ○/△/× | |
| 納得度 | ○/△/× | |
| 行動度 | ○/△/× | |
| 信頼度 | ○/△/× | |

### 総評
（3行以内で全体の印象をまとめる）

### 改善提案（優先度順）
1.
2.
3.
```
